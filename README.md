# RNA-Seq Analysis Pipeline

This repository contains an automated shell script (`run_pipeline.sh`) for processing paired-end Bulk RNA-Seq data. The pipeline takes raw FASTQ files for a specified sample, performs quality control, trims poor-quality bases, aligns the reads to a reference genome, and quantifies gene expression.

## Pipeline Overview

The pipeline executes the following steps sequentially:
1. **Directory & Dependency Validation**: Checks that all required software is installed via Conda and necessary reference files exist.
2. **Initial Quality Control (FastQC)**: Analyzes the raw reads.
3. **Read Trimming (Trimmomatic)**: Removes low-quality bases and adapter sequences.
4. **Post-trimming QC (FastQC)**: Re-evaluates read quality after trimming.
5. **Read Alignment (HISAT2)**: Aligns the trimmed reads to the provided reference genome and outputs a sorted BAM file via `samtools`.
6. **QC Aggregation (MultiQC)**: Compiles all FastQC reports into a single interactive HTML report.
7. **Read Quantification (featureCounts)**: Counts the mapped reads for genomic features (genes/exons) using a GTF annotation file.
8. **Matrix Formatting**: Cleans the resulting count matrix for downstream differential expression analysis.

## Prerequisites & Dependencies

The script now features fully automated Conda dependency management. The pipeline relies on the following tools:
- **FastQC**
- **Java** (Required for Trimmomatic)
- **HISAT2**
- **SAMtools**
- **MultiQC**
- **Subread** (Provides `featureCounts`)

If any of these packages are missing, the script will automatically create and activate a new Conda environment (`rna_seq_pipeline`) natively using the provided `environment.yml` configuration. Note that you must already have Conda installed on your machine.

### Setting up Trimmomatic

Unlike the other tools that can be installed directly into your `$PATH` (e.g., via `apt` or `conda`), the pipeline expects **Trimmomatic** to be downloaded and executed explicitly via its compiled Java `.jar` file. 

To set it up:
1. Download the latest version of [Trimmomatic](http://www.usadellab.org/cms/?page=trimmomatic).
2. Extract the downloaded zip file into your project folder.
3. Update the `TRIMMOMATIC_JAR` configuration variable in `run_pipeline.sh` to point to the exact location of the extracted `.jar` file (e.g., `Trimmomatic-0.39/trimmomatic-0.39.jar`).

### Setting up HISAT2 Reference Index

Before running the pipeline for the first time, you must build an index of your reference genome (`.fa` or `.fasta`) for HISAT2. **You only need to do this once per reference genome.**

To build the index, run the following command in your terminal:

```bash
# Example indexing for human chromosome 22 using 6 threads (adjust paths as necessary)
hisat2-build -p 6 references/chr22.fa references/index/chr22.fa
```

Make sure the output prefix (e.g., `references/index/chr22.fa`) matches the `REFERENCE_INDEX` path defined in `run_pipeline.sh`.

## Directory Structure

By default, the script expects your project directory to be organized as follows (though paths can be modified in the script's configuration variables):

```text
RNA_pipeline_update/
├── Reads/                      # Input directory containing paired raw reads 
├── references/                 # Reference genome and annotation files
│   ├── index/                  # HISAT2 pre-built indices (e.g., chr22.fa)
│   └── gtf/                    # Genome annotation GTF (e.g., chr22.gtf)
├── Trimmomatic-0.39/           # Directory containing the trimmomatic-0.39.jar file
├── environment.yml             # Conda environment definition
├── run_pipeline.sh             # Main executable script
└── output/                     # Output base directory (auto-generated by the script)
    └── {SAMPLE}/               # Output specific to the sample (e.g., demo_1)
        ├── logs/               # Execution logs for each step
        ├── qc/                 # Initial fastqc and multiqc reports
        ├── trimmed/            # Quality-trimmed FASTQ files
        │   └── qc/             # Trimmed fastqc and multiqc reports
        ├── alignments/         # Sorted BAM files
        └── quant/              # featureCounts quantification matrix
```

## Usage Configuration

Before running the pipeline, open `run_pipeline.sh` and verify the variable definitions at the top of the file:

```bash
# Base paths
PROJECT_DIR="/media/user/New_Volume/RNA_pipeline_update"
DATA_DIR="${PROJECT_DIR}/Reads"
OUTPUT_DIR="${PROJECT_DIR}/output"

REFERENCE_INDEX="${PROJECT_DIR}/references/index/chr22.fa" # HISAT2 Index base
REFERENCE_GTF="${PROJECT_DIR}/references/gtf/chr22.gtf"    # Annotation GTF

TRIMMOMATIC_JAR="${PROJECT_DIR}/Trimmomatic-0.39/trimmomatic-0.39.jar"

SAMPLE="HBR_1" # Sample prefix for processing
THREADS=6       # Number of threads/cores allocated for processing
```

## Running the Pipeline

Once configured, simply execute the script from your terminal:

```bash
chmod +x run_pipeline.sh
./run_pipeline.sh
```

The script will track the total execution time and log its progress to the console. If any step fails or reference files are missing, the pipeline will halt immediately and provide an error message. Console logs for individual tools are gracefully saved to the `output/{SAMPLE}/logs/` directory.

## Outputs

The core output of this pipeline is the expression count matrix located at:
`output/{SAMPLE}/quant/{SAMPLE}_counts_data.txt`

This count matrix includes raw gene counts and is ready to be imported into R/Python for Differential Gene Expression Analysis (e.g., using `DESeq2` or `edgeR`).
